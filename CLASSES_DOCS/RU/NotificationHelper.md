# Описание класса: NotificationHelper

## 1. Общая информация
*   **Имя класса:** `NotificationHelper`
*   **Тип:** Вспомогательный класс (Helper)
*   **Назначение:** Отвечает за создание и управление системными уведомлениями Android. Также проигрывает короткий звуковой сигнал при появлении уведомления.
*   **Взаимодействие с другими компонентами:**
    *   Создается и вызывается из `MainActivity`.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `CHANNEL_ID` | `String` | Уникальный ID канала уведомлений (обязательно для Android 8+). | `createNotificationChannel()` |
| `mediaPlayer` | `MediaPlayer` | Используется для проигрывания звука "гонга". | `playSound()` |

## 3. Методы класса
### Имя метода: `createNotificationChannel`
*   **Что делает:** Регистрирует "канал" уведомлений в системе. Современные телефоны требуют создания таких каналов, чтобы пользователь мог настраивать их важность.
*   **Когда вызывается:** Один раз при инициализации помощника.

### Имя метода: `showLoadingNotification`
*   **Что делает:** 
    1. Включает звук гонга.
    2. Создает уведомление с иконкой радио и текстом "Loading...".
    3. Выводит его в шторку уведомлений телефона.
*   **Когда вызывается:** В момент начала загрузки радиостанции.

## 4. Объяснение простыми словами
Представьте, что `NotificationHelper` — это **глашатай**. Когда происходит что-то важное (например, начинает загружаться музыка), глашатай бьет в колокол (звук) и поднимает плакат (уведомление), чтобы вы знали о происходящем, даже если экран телефона выключен.
