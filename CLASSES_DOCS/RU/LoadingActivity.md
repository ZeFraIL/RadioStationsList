# Описание класса: LoadingActivity

## 1. Общая информация
*   **Имя класса:** `LoadingActivity`
*   **Тип:** Activity (Вспомогательный экран)
*   **Назначение:** Это экран загрузки. Он появляется в тот момент, когда пользователь выбрал станцию, но звук еще не начал играть (идет буферизация).
*   **Взаимодействие с другими компонентами:**
    *   Запускается из `MainActivity` при начале воспроизведения.
    *   Закрывается автоматически или по команде из `MainActivity`.

## 2. Общая логика
1. Как только нажата кнопка "Play", этот экран перекрывает всё остальное.
2. Пользователь видит крутящийся индикатор и надпись "Loading...".
3. Внутри запускается отдельный поток (`Thread`), который ждет 3 секунды и закрывает экран.
4. Также экран умеет закрываться мгновенно, если получит сигнал "close" через `Intent`.

## 3. Методы класса
### Имя метода: `onCreate`
*   **Что делает:** Загружает дизайн и запускает "таймер".
*   **Логика таймера:** Программа "засыпает" на 3000 миллисекунд (3 секунды). После этого она вызывает команду `finish()`, чтобы убрать экран загрузки с глаз пользователя.

## 4. Объяснение простыми словами
Представьте, что `LoadingActivity` — это **зал ожидания**. Когда вы просите включить радио, вас просят немного подождать, чтобы сигнал успел дойти. Чтобы вы не смотрели в пустой экран и не думали, что приложение зависло, вам показывают этот экран-заставку.
