## 🎯 Цель проекта
*   **Какую задачу решает приложение:** Обеспечивает простой и эффективный доступ к потоковому вещанию онлайн-радиостанций по всему миру через API Radio Browser.
*   **Почему эта задача важна:** Позволяет пользователям слушать глобальный медиа-контент и музыку разных жанров (джаз, новости и т.д.) в легком мобильном формате с поддержкой фонового воспроизведения.
*   **Целевая аудитория:** Любители радио, путешественники и пользователи, интересующиеся мировыми новостями или специфическими музыкальными жанрами.
________________________________________
## 📌 Требования к приложению
### Функциональные требования
*   Получение списка топовых радиостанций через REST API.
*   Фильтрация станций по категориям/жанрам (Все, Джаз, Новости).
*   Воспроизведение потоков непосредственно в приложении.
*   Поддержка фонового воспроизведения через Android Service.
*   Отображение уведомлений во время загрузки и воспроизведения.
### Нефункциональные требования
*   **Производительность:** Быстрая загрузка списка станций благодаря асинхронным сетевым вызовам.
*   **Удобство использования:** Простой, интуитивно понятный интерфейс с минимальным количеством переходов.
*   **Надёжность:** Надежная обработка сетевых сбоев и ошибок буферизации потока.

## 🧠 Общая архитектура
*   **Выбранный подход:** MVC (Model-View-Controller), адаптированный под Android (Activity-centric).
*   **Почему выбран именно он:** Выбран за простоту и прямолинейность для утилитарного приложения, что обеспечивает минимальные накладные расходы.
*   **Основные компоненты системы:**
    *   **Activities (Активности):** Контроллеры интерфейса (`MainActivity`, `GenreActivity`, `LoadingActivity`).
    *   **Service (Сервис):** `RadioService` для фонового воспроизведения аудио.
    *   **Model (Модель):** POJO-класс `RadioStation` для представления данных.
    *   **Helper (Помощник):** `NotificationHelper` для взаимодействия с системными уведомлениями.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
*   **Роль:** Главный функциональный узел.
*   **Ответственность:** Отображает список станций, обрабатывает действия пользователя, инициирует вызовы API и управляет локальным воспроизведением.
*   **Основные методы:**
    *   `onCreate()`: Инициализация интерфейса и получение данных.
    *   `fetchRadioStations()`: Выполняет асинхронный запрос OkHttp к API Radio Browser.
    *   `playRadio()`: Управляет экземпляром MediaPlayer и состоянием UI.
### 📌 Класс: RadioService
*   **Роль:** Компонент фонового выполнения.
*   **Ответственность:** Поддерживает воспроизведение аудио, даже когда интерфейс приложения не активен.
*   **Основные методы:**
    *   `onStartCommand()`: Получает URL потока и запускает воспроизведение.
    *   `onDestroy()`: Освобождает ресурсы при остановке сервиса.

## 🔄 Диаграмма работы приложения
1.  **Запуск:** Пользователь выбирает жанр в `GenreActivity`.
2.  **Загрузка:** `MainActivity` запрашивает данные JSON из API.
3.  **Выбор:** Пользователь нажимает на станцию в `ListView`.
4.  **Буферизация:** Появляются `LoadingActivity` и высокоприоритетное уведомление, пока поток готовится.
5.  **Прослушивание:** Запускается аудио; пользователь может переключиться в режим фонового сервиса или остановить воспроизведение.

## ⚙️ Работа с потоками
*   **Механизмы:**
    *   `Thread` (для таймера загрузки).
    *   `OkHttp Async Callbacks` (для сетевых запросов).
    *   `runOnUiThread()` (для обновления UI из фоновых потоков).
*   **Почему выбран этот способ:** Гарантирует, что основной поток UI никогда не блокируется во время сетевых операций или буферизации.
*   **Предотвращение:**
    *   *ANR:* Весь ввод-вывод вынесен за пределы основного потока.
    *   *Утечки памяти:* Ресурсы MediaPlayer и Service явно освобождаются в `onDestroy()`.
________________________________________
## 💾 Работа с данными
