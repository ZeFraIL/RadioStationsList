## 🎯 מטרת הפרויקט
*   **הבעיה שהאפליקציה פותרת:** מספקת דרך פשוטה ויעילה להאזנה לתחנות רדיו מקוונות מרחבי העולם באמצעות ה-Radio Browser API.
*   **חשיבות המשימה:** מאפשרת למשתמשים לגשת לתוכן מדיה גלובלי ולז'אנרים מוזיקליים שונים (ג'אז, חדשות וכו') בפורמט נייד קל משקל עם תמיכה בהשמעה ברקע.
*   **קהל יעד:** חובבי רדיו, מטיילים ומשתמשים המחפשים חדשות עולמיות או ז'אנרים מוזיקליים ספציפיים.
________________________________________
## 📌 דרישות האפליקציה
### דרישות פונקציונליות
*   קבלת רשימת תחנות רדיו מובילות דרך REST API.
*   סינון תחנות לפי קטגוריה/ז'אנר (הכל, ג'אז, חדשות).
*   השמעת שידורי רדיו ישירות בתוך האפליקציה.
*   תמיכה בהשמעה ברקע באמצעות Android Service.
*   הצגת התראות (Notifications) בזמן טעינה והשמעה.
### דרישות לא פונקציונליות
*   **ביצועים:** טעינה מהירה של רשימת התחנות באמצעות קריאות רשת אסינכרוניות.
*   **חווית משתמש:** ממשק פשוט ואינטואיטיבי עם מינימום שלבי ניווט.
*   **אמינות:** טיפול חזק בשגיאות רשת ובעיות אגירה (buffering) של השידור.
________________________________________
## 🧠 ארכיטקטורה כללית
*   **גישה נבחרת:** MVC (Model-View-Controller) מותאם לאנדרואיד (ממוקד Activity).
*   **סיבת הבחירה:** נבחרה בשל הפשטות והישירות שלה עבור אפליקציה שימושית (utility), המבטיחה תקורה (overhead) נמוכה.
*   **רכיבי המערכת העיקריים:**
    *   **Activities:** בקרי ממשק משתמש (`MainActivity`, `GenreActivity`, `LoadingActivity`).
    *   **Service:** `RadioService` לביצוע השמעת אודיו ברקע.
    *   **Model:** מחלקת `RadioStation` לייצוג הנתונים.
    *   **Helper:** `NotificationHelper` לאינטראקציות עם המערכת.
________________________________________
## 🧩 תרשים UML
`[GenreActivity] -> [MainActivity]`  
`[MainActivity] -> [RadioStation] (Model)`  
`[MainActivity] -> [RadioService] (Background Execution)`  
`[MainActivity] -> [NotificationHelper] -> [Android System Notification]`  
`[MainActivity] -> [LoadingActivity] (Status UI)`

## 🧩 תיאור מפורט של מחלקות
### 📌 מחלקה: MainActivity
*   **תפקיד:** המרכז הפונקציונלי הראשי.
*   **אחריות:** מציגה את רשימת התחנות, מטפלת באינטראקציות משתמש, יוזמת קריאות API ושולטת בהשמעה מקומית.
*   **מתודות עיקריות:**
    *   `onCreate()`: אתחול ממשק המשתמש וקבלת נתונים.
    *   `fetchRadioStations()`: מבצעת בקשת OkHttp אסינכרונית ל-Radio Browser API.
    *   `playRadio()`: מנהלת את מופע ה-MediaPlayer ומצב ממשק המשתמש.
### 📌 מחלקה: RadioService
*   **תפקיד:** רכיב ביצוע ברקע.
*   **אחריות:** שומרת על השמעת אודיו גם כאשר ממשק המשתמש אינו בקדמת הבמה.
*   **מתודות עיקריות:**
    *   `onStartCommand()`: מקבלת את כתובת ה-URL של השידור ומתחילה בהשמעה.
    *   `onDestroy()`: משחררת משאבים כאשר השירות נעצר.

## 🔄 זרימת עבודת האפליקציה
1.  **הפעלה:** המשתמש בוחר ז'אנר ב-`GenreActivity`.
2.  **טעינה:** `MainActivity` מבקשת נתוני JSON מה-API.
3.  **בחירה:** המשתמש לוחץ על תחנה ב-`ListView`.
4.  **טעינה זמנית:** `LoadingActivity` והתראה בעדיפות גבוהה מופיעים בזמן שהשידור מתכונן.
5.  **האזנה:** האודיו מתחיל; המשתמש יכול לעבור למצב שירות רקע או להפסיק את ההשמעה.
________________________________________
## 🎨 ניתוח UI/UX
*   **בחירת עיצוב:** פריסה ליניארית (LinearLayout) עם כפתורים ברורים וגדולים ורשימה סטנדרטית למוכרות.
*   **עקרונות:**
    *   *פשטות:* מספר מינימלי של לחיצות להגעה לתוכן.
    *   *לוגיקה:* זרימה מבחירת ז'אנר לרשימת תחנות.
    *   *נגישות:* ניגודיות גבוהה וגודל טקסט גדול (35sp לכותרת הז'אנר).
________________________________________
## ⚙️ עבודה עם תהליכונים (Threads)
*   **מנגנונים בשימוש:**
    *   `Thread` (עבור טיימר הטעינה).
    *   `OkHttp Async Callbacks` (עבור בקשות רשת).
    *   `runOnUiThread()` (עבור עדכון ממשק המשתמש מתהליכוני רקע).
*   **סיבת הבחירה:** מבטיחה שתהליכון ממשק המשתמש הראשי לעולם לא ייחסם במהלך פעולות רשת או אגירה.
*   **מניעה:**
    *   *ANR:* כל פעולות ה-I/O מבוצעות מחוץ לתהליכון הראשי.
    *   *דליפות זיכרון:* משאבי MediaPlayer ושירות משוחררים במפורש ב-`onDestroy()`.
