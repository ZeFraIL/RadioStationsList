# Описание класса: RadioStation

## 1. Общая информация
*   **Имя класса:** `RadioStation`
*   **Тип:** Обычный класс (POJO / Модель данных)
*   **Назначение:** Этот класс служит контейнером для хранения информации о конкретной радиостанции. Он не содержит сложной логики, а лишь хранит данные.
*   **Взаимодействие с другими компонентами:**
    *   Используется в `MainActivity` для хранения списка станций.
    *   Создается при разборе данных (JSON), полученных из интернета.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `name` | `String` | Официальное название радиостанции. | Конструктор, геттеры |
| `streamUrl` | `String` | Прямая ссылка на аудиопоток. | Конструктор, геттеры |
| `bitrate` | `String` | Качество вещания (битрейт). | Конструктор |
| `homepage` | `String` | Ссылка на сайт станции. | Конструктор |

## 3. Методы класса
### Имя метода: `RadioStation` (Конструктор)
*   **Тип:** `public`
*   **Что делает метод:** Принимает все данные о станции и записывает их во внутренние переменные объекта.
*   **Когда вызывается:** В `MainActivity`, когда приложение находит новую станцию в ответе от сервера.

### Методы `getName`, `getStreamUrl` и др. (Геттеры)
*   **Тип:** `public`
*   **Что делают:** Позволяют другим частям программы "прочитать" данные, хранящиеся внутри объекта, так как сами переменные закрыты (private).

## 4. Общая логика
Класс работает как обычная папка с документами. Он просто хранит информацию в структурированном виде, чтобы программисту было удобно обращаться к полям `name` или `url`, не путаясь в строках текста.

## 5. Объяснение простыми словами
Представьте, что `RadioStation` — это **Визитная карточка**. На этой карточке написано название станции, её "номер телефона" (ссылка для прослушивания) и сайт. Вместо того чтобы носить гору разрозненных бумажек, приложение складывает всю информацию на одну аккуратную визитку.
