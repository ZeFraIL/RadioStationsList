# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность/Экран)
*   **Назначение:** Это главный экран приложения. Он отвечает за получение списка радиостанций из интернета, отображение их пользователю и управление воспроизведением аудио (локально или через фоновый сервис).
*   **Взаимодействие с другими компонентами:**
    *   Получает данные (жанр) от `GenreActivity` через `Intent`.
    *   Запускает `LoadingActivity` для отображения прогресса пользователю.
    *   Использует `RadioService` для фонового воспроизведения.
    *   Использует `NotificationHelper` для системных уведомлений.
    *   Использует `RadioStation` как модель данных.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `RADIO_BROWSER_API` | `String` | Константа с URL-адресом для API радиостанций. | `fetchRadioStations()` |
| `listView` | `ListView` | Элемент интерфейса для отображения списка станций. | `initComponents()` |
| `radioStations` | `ArrayList<RadioStation>` | Список объектов с подробными данными о каждой станции. | `fetchRadioStations()` |
| `stationNames` | `ArrayList<String>` | Список названий станций для отображения в `ListView`. | `fetchRadioStations()` |
| `mediaPlayer` | `MediaPlayer` | Системный компонент для воспроизведения аудио. | `playRadio()`, `stopRadio()` |
| `what_genre` | `String` | Хранит жанр, выбранный пользователем на предыдущем экране. | `initComponents()` |

## 3. Методы класса
### Имя метода: `fetchRadioStations`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** Нет
*   **Что делает метод:**
    1.  Создает `OkHttpClient` для работы с сетевым запросом.
    2.  Формирует `Request` (запрос) по адресу API.
    3.  Асинхронно (в фоновом потоке) выполняет запрос.
    4.  При успехе: Разбирает JSON-ответ в список объектов `RadioStation`.
    5.  Фильтрует станции по выбранному жанру `what_genre`.
    6.  Обновляет список на экране через `runOnUiThread`.
*   **Когда вызывается:** Автоматически при запуске активности в методе `initComponents()`.

### Имя метода: `playRadio`
*   **Тип метода:** `private`
*   **Параметры:** `streamUrl` (ссылка на поток), `stationName` (название станции).
*   **Что делает метод:**
    1.  Сбрасывает состояние `mediaPlayer`, если он уже используется.
    2.  Показывает уведомление о загрузке и открывает `LoadingActivity`.
    3.  Устанавливает источник данных (ссылку) для плеера.
    4.  Вызывает `prepareAsync()` для буферизации (подготовки) потока без зависания экрана.
    5.  После подготовки запускает музыку и закрывает экран загрузки.
*   **Когда вызывается:** При нажатии пользователем "Да" в диалоговом окне выбора станции.

## 4. Жизненный цикл
*   **`onCreate()`**: Вызывается при создании экрана. Устанавливает макет (`activity_main`) и инициализирует компоненты.
*   **`onDestroy()`**: Вызывается при закрытии приложения. Освобождает ресурсы `mediaPlayer`, чтобы они не занимали память.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `ListView` (список), кнопки "Stop", "Play (service)", текстовое поле жанра.
*   **Связь с кодом:** Через `findViewById(R.id.имя)`.
*   **События:** Нажатие на элемент списка открывает диалог подтверждения; нажатие кнопок управляет плеером или сервисом.

## 6. Взаимодействие с другими компонентами
Использует **Intents** для получения жанра и запуска сервиса. Работает с внешней библиотекой **OkHttp** для запросов к серверу `radio-browser.info`.

## 7. Общая логика
Приложение запускается, узнает выбранный жанр и сразу "звонит" на сервер за списком станций. Получив список, оно показывает его пользователю. При клике на станцию начинается процесс буферизации и воспроизведения.

## 8. Объяснение простыми словами
Представьте, что `MainActivity` — это **умный радиоприемник с экраном**.
1. Он смотрит на "карточку с жанром", которую вы ему дали.
2. Он звонит в "справочную" (API) и просит список станций для этого жанра.
3. Он записывает их в блокнот (список) на экране.
4. Когда вы тыкаете в название, он начинает "ловить волну" (загружать данные) и включает звук. Если вы уходите, он передает управление **фоновому помощнику** (Service), чтобы музыка не прерывалась.
